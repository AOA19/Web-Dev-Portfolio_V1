Project Purpose and Goal

This project was completed as a Frontend Mentor challenge and built with HTML, CSS, and Vanilla JavaScript. The purpose of this challenge was to recreate this tip calculator app and get it as close to the provided JPG design file as possible.

I chose this project because for this calculator to function, JavaScript is essential. While building this calculator app I was able to test my knowledge on JavaScript operators.

Problems and Thought Process

This project had its challenges one particular difficulty I had was getting the tip per-person calculation to work properly. The issues with the tip calculation ultimately affected the total bill per person calculation.

The value of the tip per-person would come back undefined whenever the tip button was clicked or a custom tip was inputted. I realized the separate function created with a for loop that would loop through each tip button to get the value, wasn’t working inside the function. So I placed the for loop statement inside of the function that calculates the total bill. Once I made this change, the calculation worked properly. This change also fixed other issues I was having with my code.

Lessons Learned

I was able to learn a lot about JavaScript operators while working on this project. I learned how useful and flexible JavaScript operators are when used in code.

Something else that I was able to learn about was making the reset button accessible. I learned that when a button is disabled using the disabled attribute, it takes away the keyboard focus on that button. Which makes it difficult for users that use their keyboards to navigate through the web. So instead of using the disabled attribute on the button, use the aria-disabled attribute. This information was very useful for me because I want to make sure any project I work on is web accessible.
